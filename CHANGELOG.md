## v0.0.00001 - Build 27 - 2026-07-22

■更新履歴（Readme / ChangeLog 用）

- 骨格検出を「人物検知（シルエット）領域内で実行」する方式へ変更した。人物検知の方が安定しているというフィードバックを受け、人物ごとに領域をクロップして骨格検出を行い、検出関節をその人物のマスク内（+輪郭緩衝）に限定する。

■更新履歴

- `VisionPoseEstimator` を人物領域クロップ方式へ刷新: 人物矩形+15%マージンをクロップして `VNDetectHumanBodyPoseRequest` を人物ごとに実行。クロップ内で複数骨格が出た場合はマスク内関節数が最多のものを採用し、マスク外（緩衝0.03含む）の関節は隣接人物の写り込みとみなし除外。
- クロップにより人物が画面内で大きく写るため、関節検出の精度自体も向上する（小さく写る人物で特に有効）。
- `PersonMaskSampler.containsNear(x:y:radius:)`（緩衝付き内包判定）を追加（テスト追加）。
- Build 26の全体一括検出+貪欲マッチングは本方式に置き換え（対応付け誤りの余地自体がなくなる）。

## v0.0.00001 - Build 26 - 2026-07-22

■更新履歴（Readme / ChangeLog 用）

- バグ修正: 2人の人物矩形が大きく重なる構図で、骨格と人物の対応付け（人物Nグループへの割り振り）が誤る問題を修正した。シルエットマスク内に入る関節数を主基準とする貪欲マッチングへ変更。
- バグ修正: 保持している骨格関節がボーン描画閾値の不整合により描画されないことがある問題を修正した（保存閾値0.1に描画閾値を統一）。

■更新履歴

- `PersonMaskSampler` を新設。人物シルエットマスクを128x128へ縮小サンプリングし、関節の内包判定を高速に行う（単体テスト追加）。
- `VisionPoseEstimator` の対応付けを刷新: 全（人物×骨格）ペアについて「マスク内関節数（主基準）+中心距離（タイブレーク）」でスコアリングし、良いペアから貪欲に割当。マスクを持つ人物に関節が1つも入らない骨格は誤対応とみなし割り当てない。
- `NormalizedRect.contains(x:y:)` を追加。

■補足（仕様説明）

- シルエット（セグメンテーション）と骨格（姿勢推定）は別のVision APIのため、マスクが取れても骨格関節が取れないことがある（四つん這い・密着などの姿勢では関節信頼度が低く、検出できた関節のみ描画する正直表示によりボーンが少なくなる）。骨格検出率の根本改善はPhase 2本命（RTMPose等の高精度姿勢推定への差し替え検討を含む）で対応。
