# Changelog

## v0.0.00001 - Build 16 - 2026-07-22

■更新履歴（Readme / ChangeLog 用）

- 仕様確定: 人物を検出できない画像では偽の固定矩形を表示せず、候補0件として「手動追加してください」と案内するようにした（Build 14で一時復活させた互換フォールバックを廃止）。

■更新履歴

- 経緯: Phase 1以前の「常に1件返る人物矩形」は実際には検出結果ではなく固定比率の想定枠だったことをユーザーへ報告し、「検出していない枠線を表示しない＝正確な検知内容の表示を優先」との方針指示を受けて確定。
- `VisionPersonDetector` の検出0件時フォールバック矩形を削除（インスタンスマスク→人物矩形の2段検出のみ。0件は0件として返す）。
- 候補生成で人物0件・候補0件のとき、ステータスへ「人物を検出できませんでした（候補0件）。ドラッグで手動追加してください」を表示。
- テストを新方針へ更新（人物なし画像で検出0件・候補0件を検証）。

■既知の問題（未修正・継続観測）

- 漫画・イラストは実写学習のVisionでは検出率が低く、候補0件になりやすい。手動ROI+ローカル学習（Phase 4）で補いつつ、根本改善はPhase 2本命（漫画+実写データでのファインチューニング）で対応予定。

## v0.0.00001 - Build 15 - 2026-07-22

■更新履歴（Readme / ChangeLog 用）

- 画像ごとに編集状態（ROI・検出レイヤ・アンドゥ履歴・モザイク表示状態）をセッション内で保持し、ライブラリで画像を切り替えて戻ってきても編集を継続できるようにした。
- 「モザイク表示」チェックボックスを追加し、モザイクの解除/再適用をワンクリックで切替できるようにした。解除中はROIが保持され再編集可能。

■更新履歴

- `PerImageEditState` によるセッション内キャッシュ（直近8画像、LRU方式でメモリ抑制）を追加。画像切替時に自動退避し、戻ってきたときに復元する。
- 「モザイク表示」トグル: ONで現在のROIとマスク生成方式による未保存プレビューを表示、OFFで元画像+ROI表示。編集開始・候補生成・ROIクリアで自動解除され、アンドゥ/リドゥとも状態同期する。
- ライブラリ画像を開く際、作業画像を常に元画像とし、加工後画像はモザイク表示側へ読み込む方式に変更。加工済み画像でも解除→再編集→再適用が可能になった。
- 仕様変更: 「元画像を開く」でも保存済みROIを復元するようにした（再編集の起点。まっさらに戻す場合は「ROIクリア」を使用）。

■既知の問題（未修正・継続観測）

- 編集状態のセッション内キャッシュは直近8画像まで（超過分は古い順に破棄。保存済みデータはライブラリに残るため失われない）。
- アプリ再起動でセッション内キャッシュは消える（保存済みROI・加工後画像はライブラリから復元される）。

## v0.0.00001 - Build 14 - 2026-07-22

■更新履歴（Readme / ChangeLog 用）

- バグ修正: Phase 1実装後、漫画・イラスト等でVisionが人物を検出できない画像において自動認識が全く動作しなくなっていた問題を修正した（マスク生成方式の選択に関わらず候補ROIが0件になる症状）。

■更新履歴

- 原因: Phase 1以前は検出成否に関わらず固定比率の人物想定矩形を必ず1件返していたが、Phase 1で「検出0件なら空配列」へ変更したため、実写学習のVision（インスタンスマスク・人物矩形とも）が反応しない漫画画像で候補が全滅していた。
- 修正: `VisionPersonDetector.detectPersons` に検出0件時の従来互換フォールバック（中央の人物想定矩形1件）を復活。下流のヒューリスティックROI生成と手動調整の起点を常に確保する。
- 併せて `VNDetectHumanRectanglesRequest` に revision 2 を明示指定（`upperBodyOnly=false` の全身矩形はrevision 2でのみ有効なため）。
- 回帰防止テストを2件追加（人物なし画像でのフォールバック矩形、パイプライン全体で候補が必ず1件以上）。

## v0.0.00001 - Build 13 - 2026-07-22

■更新履歴（Readme / ChangeLog 用）

- 自動認識Phase 4: モザイク範囲選択のローカル学習を追加した。採用/削除したROIを収集し、選択位置の頻度・サイズ・パッチの近似画像参照（知覚ハッシュ）で候補生成の精度を継続的に改善する。学習データはすべてローカル保存で外部送信しない。

■更新履歴

- `LearningEngine` を新設（`Application Support/newMosaic/Learning/`）。保存時に採用ROI=正例、削除された自動候補=負例として記録し、パッチPNG（最大256px）も収集（将来の部位検出モデル学習データを兼ねる）。
- カテゴリ別に人物相対8x8位置頻度グリッド・平均サイズ・dHash知覚ハッシュを蓄積。候補生成時に位置頻度ブースト（最大+0.3）、正例近似で+0.2/負例近似で−0.25の信頼度調整、高頻度位置への `learned-prior` ROI自動提案（最大8件）を行う。
- 近似画像参照は「Google画像検索型の類似検索」をローカル完結のdHashハミング距離近傍探索で代替（外部サービス不使用・プライバシー準拠）。
- 処理負荷: 集計は保存時のみ、推論時はグリッド参照+線形ハッシュ走査で数千サンプルまで1ms級。
- `Mosaic/DEBUG_LOG_INVENTORY.md` に学習ストアの保存先・個人情報の有無・削除方法を追記。

■既知の問題（未修正・継続観測）

- 学習効果はサンプル5件以上から発現する設計（それ未満は従来動作と同一）。
- パッチ収集はモザイク対象範囲の画像断片を含むため、`Learning/` フォルダの取り扱いに注意（削除すれば完全消去）。

## v0.0.00001 - Build 11 - 2026-07-22

■更新履歴（Readme / ChangeLog 用）

- 自動認識Phase 2（暫定）: 自動候補ROIをVisionの顕著領域検出で精密化する `SaliencyCandidateDetector` を追加し、候補生成パイプラインの既定にした。

■更新履歴

- `SaliencyCandidateDetector` を新設。自動候補（pose-chest / pose-groin / heuristic-lower-body）の周辺1.8倍領域をクロップし `VNGenerateObjectnessBasedSaliencyImageRequest` を実行、元ROIとの重なり率0.2以上・面積4倍以内の顕著領域があればROIをそこへ精密化する。整合しない場合は元ROIを維持（再現率優先）。
- `NormalizedRect` に `area` / `expanded(scale:)` / `intersection(_:)` を追加。
- モデル選定: 学習済み部位検出モデル（Phase 2本命）はデータセットとライセンス確認が必要なため、追加コスト0のVisionサリエンシーを暫定採用。プロトコル境界は維持しており、部位検出モデル取得後に `CandidateDetecting` 実装を差し替えるだけで移行できる。

■既知の問題（未修正・継続観測）

- サリエンシーは「目立つ物体」への吸着であり部位の意味分類ではない。カテゴリ別の真の部位検出はデータセット整備後に対応。

## v0.0.00001 - Build 10 - 2026-07-22

■更新履歴

- `VisionPersonDetector` を `VNGeneratePersonInstanceMaskRequest`（最大4人、インスタンス別マスク）ベースに書き換え。ラベルマップから人物ごとの外接矩形を算出し、シルエットマスクCGImageも取得。失敗時は `VNDetectHumanRectanglesRequest` へフォールバック。

- `SensitiveROIGenerator` を関節ベースの解剖学的プライア方式に書き換え（関節なしは従来比率へフォールバック、既存動作維持）。
- モデル選定: Apple Vision標準API（追加コスト0・完全ローカル・macOS 14前提で可用性100%）を採用。

■既知の問題（未修正・継続観測）

- 鼠径部ROIのカテゴリは性別分類器が未導入のため `.other`（男女判別はPhase 2で対応予定）。
- 人物インスタンスマスクは最大4人（Vision制約）。5人以上は矩形検出のみに縮退。
- 検証画像が漫画の場合、Vision（実写学習）の検出率は実写より低い（ドメインギャップ、Phase 2で対応検討）。

## v0.0.00001 - Build 4 - 2026-07-22

■更新履歴（Readme / ChangeLog 用）

- ウィンドウタイトルにリリース番号（v<MARKETING_VERSION> (beta Build <CURRENT_PROJECT_VERSION>)）を表示するようにした。
- モザイク対象ROIに矩形／楕円の形状選択を追加し、直前に使ったサイズを形状ごとに記憶してワンクリックで同サイズ追加できるようにした。既存ROIは選択後にサイズ変更・形状変更ができる。
- 人物検出・骨格検出の中間結果を画像上に半透明レイヤとして個別表示できるようにした。

■更新履歴

- `MosaicROI` に `shape: ROIShape`（矩形／楕円）を追加。既存データ（shapeキー無し）は `.ellipse` として後方互換デコードする。
- `MosaicEngine` に矩形マスク生成を追加し、ROIの形状に応じてモザイク描画を切り替える。
- `StaticImageMosaicPipeline.generateDetailedCandidates(for:)` を追加し、人物検出・骨格ヒント・ROIの中間結果を取得可能にした。
- `ImageCanvasView` にROI選択・四隅リサイズハンドル・形状別プレビュー描画・検出レイヤ描画を追加した。

■更新履歴（画面・機能別）

- macOS: ツールバー2段目に「追加形状（矩形／楕円）」「人物検出レイヤ」「骨格検出レイヤ」を追加。
- macOS: キャンバス上でROIをクリック選択→四隅ハンドルでリサイズ、形状ボタンで形状変更が可能。
- macOS: 記憶済みサイズがある形状は、空き領域をクリックするだけでそのサイズのROIを追加できる。

■既知の問題（未修正・継続観測）

- 自動検出（`PersonDetector` / `PoseEstimator` / `ROIGenerator` / `CandidateDetector`）は固定比率の算術ヒューリスティックのままで、画像内容に応じた実検出は未実装。
- `SegmentEngine`（`EllipseSegmentEngine`）はパイプラインから未接続で、実際のマスク生成は `MosaicEngine` が別途重複実装している。
- ROIの移動（ドラッグでの位置変更）とキーボードでの個別削除は未対応（クリア全消去のみ）。
- 形状ごとの記憶サイズはセッション内のみでアプリ再起動後は保持されない。
